# Cognitive Accessibility

Cognitive accessibility covers users with dyslexia, ADHD, memory impairments, autism, intellectual disabilities, and situational limitations (fatigue, unfamiliar language). WCAG 2.2 added several criteria with cognitive users in mind.

## Plain language

Keep sentences short, use common words, and put the main point first. Avoid idioms and metaphors that don't translate. WCAG 3.1.5 Reading Level (AAA) suggests a lower secondary education reading level for general-audience content.

## Consistency

WCAG 3.2.3 Consistent Navigation and 3.2.4 Consistent Identification require that navigation and labels stay consistent across a site. Switching between "Log in", "Sign in", and "Login" for the same action adds cognitive load.

## Timing

WCAG 2.2.1 Timing Adjustable requires that any time limit can be turned off, adjusted, or extended (except where real-time events or essential security concerns apply). Auto-logout after idle time must warn the user and offer an extension.

## Memory and authentication

WCAG 3.3.8 Accessible Authentication (Minimum, AA, new in 2.2) forbids authentication steps that require cognitive function tests (memorizing, transcribing, solving puzzles) without an alternative such as a password manager, copy-paste support, or biometrics.
